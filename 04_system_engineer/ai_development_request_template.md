## 2. 환경 정의

### 환경별 구성

| 항목           | dev (Development)       | qa (Quality Assurance)  | stg (Staging)           | prd (Production)        |
|----------------|-------------------------|-------------------------|-------------------------|-------------------------|
| 용도           | 개발/테스트             | QA 검증                 | 운영 동일 검증          | 실서비스                |
| 서버 수        | 1                       | 2                       | 2                       | 4                       |
| 스펙           | c6i.xlarge (4C/8G)      | c6i.2xlarge (8C/16G)    | c6i.4xlarge (16C/32G)   | c6i.4xlarge (16C/32G)   |
| DB             | MySQL 단독              | MySQL 단독              | MySQL Replication       | MySQL Multi-AZ          |
| 네트워크       | 10.10.0.0/16            | 10.20.0.0/16            | 10.30.0.0/16            | 10.0.0.0/16             |
| 도메인         | dev.game.internal       | qa.game.internal        | stg.game.internal       | game.example.com        |
| 배포 방식      | 수동                    | 수동                    | CI/CD (승인 필요)       | CI/CD (승인 필요)       |
| 모니터링       | 기본                    | 기본                    | zabbix                  | zabbix + grafana        |
| 백업           | 없음                    | 일 1회                  | 일 1회                  | 일 4회 + 실시간         |
### 환경별 흐름

```
+-------+        +-------+        +-------+        +-------+
|  dev  |------->|  qa   |------->|  stg  |------->|  prd  |
+-------+        +-------+        +-------+        +-------+
  개발자           QA 팀            운영 팀           운영 팀
  자유 배포        테스트 완료       prd 동일 검증     최종 배포
                   후 승인          후 승인           후 승인
```

### 핫픽스 흐름 (긴급)

```
+-------+        +-------+        +-------+
|  dev  |------->|  stg  |------->|  prd  |
+-------+        +-------+        +-------+
  긴급 수정        최소 검증         즉시 배포
  (QA 생략)        (30분 이내)       (승인 필수)

  ※ 핫픽스 후 반드시 qa 환경에 소급 반영
```

### 환경별 접근 권한

| 역할           | dev   | qa    | stg   | prd   |
|----------------|-------|-------|-------|-------|
| 개발자         | ✅    | ✅    | ❌    | ❌    |
| QA 엔지니어   | ✅    | ✅    | ✅    | ❌    |
| 시스템 엔지니어| ✅    | ✅    | ✅    | ✅    |
| DBA            | ❌    | ❌    | ✅    | ✅    |

### 환경별 네이밍 규칙

```
[env]-[category]-[service]-[detail]

예시:
  dev-app-game-01        prd-app-game-01
  dev-db-mysql-01        prd-db-mysql-01
  dev-net-vpc-main       prd-net-vpc-main
  dev-sg-app-game        prd-sg-app-game
```

## 5. 요구사항

### 기능 요구사항

| #   | 요구사항              | 우선순위 | 적용 환경          | 비고     |
|-----|-----------------------|----------|--------------------|----------|

| F02 | 서버 시작/중지        | 필수     | dev/qa/stg/prd     | 승인 필요 |
| F03 | 로그 조회             | 선택     | stg/prd            |          |

### 비기능 요구사항

| #   | 요구사항              | 기준              | 비고                    |
|-----|-----------------------|-------------------|-------------------------|
| N01 | 실행 시간             | 5분 이내          |                         |

| N03 | 로깅                  | /var/log/ 에 기록 |                         |
| N04 | 환경 분리             | 환경별 설정 파일  | env.conf 또는 .env 방식 |

| 항목              | dev/qa             | stg/prd                |

| 데이터 마스킹     | 불필요             | 필수 (ISMS)            |
| 승인 프로세스     | 불필요             | 필수                   |

## 8. 테스트 기준

| #   | 시나리오              | 예상 결과         | 확인 방법         | 테스트 환경 |
|-----|-----------------------|-------------------|-------------------|-------------|
| T01 | 정상 실행             | 성공 로그 기록    | 로그 파일 확인    | dev         |
| T02 | 디스크 부족 상황      | 알림 발송         | slack 수신 확인   | qa          |
| T03 | 권한 없는 사용자 실행 | 에러 메시지 출력  | exit code 확인    | qa          |
| T04 | prd 동일 환경 검증    | dev/qa와 동일 결과| stg에서 실행      | stg         |

## 9. 배포 계획

### 일반 배포

| 단계 | 환경 | 작업 내용              | 승인자         | 롤백 방안              |
|------|------|------------------------|----------------|------------------------|
| 1    | dev  | 개발 및 단위 테스트    | -              | -                      |
| 2    | qa   | QA 검증                | QA 팀          | 이전 버전 복원         |
| 3    | stg  | prd 동일 환경 검증     | 운영 팀        | 이전 버전 복원         |
| 4    | prd  | 실서비스 적용          | 운영 팀 + PL   | 즉시 롤백 (스크립트)   |

### 핫픽스 배포

| 단계 | 환경 | 작업 내용              | 승인자         | 롤백 방안              |
|------|------|------------------------|----------------|------------------------|
| 1    | dev  | 긴급 수정 및 테스트    | -              | -                      |
| 2    | stg  | 최소 검증 (30분 이내)  | 운영 팀        | 이전 버전 복원         |
| 3    | prd  | 즉시 적용              | 운영 팀 + PL   | 즉시 롤백 (스크립트)   |
| 4    | qa   | 소급 반영 및 검증      | QA 팀          | -                      |

### 배포 전 체크리스트

- [ ] 수정 코드 리뷰 완료
- [ ] dev 환경 테스트 완료
- [ ] stg 환경 최소 검증 완료
- [ ] 롤백 스크립트 준비 완료
- [ ] 운영 팀 승인 완료

### 배포 후 체크리스트

- [ ] 서비스 정상 동작 확인

- [ ] 에러 로그 확인
- [ ] qa 환경 소급 반영 완료
- [ ] 장애 보고서 (Post-Mortem) 작성
- [ ] 재발 방지 대책 수립
