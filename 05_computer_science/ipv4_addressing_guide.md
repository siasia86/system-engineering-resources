## 1. IP 주소 기본 구조

### IPv4 주소 구성

```
192.168.1.100
├── Octet1: 192
├── Octet2: 168
├── Octet3: 1
└── Octet4: 100

Each octet: 0 ~ 255 (8 bits = 1 byte)
Total: 32 bits (4 bytes)
```

- 각 옥텟: 0~255 (8비트)
- 전체: 32비트

### 네트워크 영역 vs 호스트 영역

```
IP Address = Network Part + Host Part

192.168.1.100/24
├── Network: 192.168.1  (identifies network)
└── Host:    100        (identifies device)
```

| 영역          | 역할                          |
|---------------|-------------------------------|
| 네트워크 영역 | 어느 네트워크에 속하는지 식별 |
| 호스트 영역   | 네트워크 내 특정 장치 식별    |

비유: `192.168.1` = 서울시 강남구, `100` = 123번지

## 3. 서브넷 마스크

### 개념

```
Subnet mask: separates network and host parts

Binary:
11111111.11111111.11111111.00000000
└──────────── 1 (network) ────────┘└── 0 (host) ──┘

Decimal: 255.255.255.0
CIDR:    /24
```

### 주요 서브넷 마스크

| CIDR | 서브넷 마스크   | 호스트 비트 | 전체 IP    | 사용 가능 IP    |
|------|-----------------|-------------|------------|-----------------|
| /8   | 255.0.0.0       | 24          | 16,777,216 | 16,777,214      |
| /12  | 255.240.0.0     | 20          | 1,048,576  | 1,048,574       |
| /16  | 255.255.0.0     | 16          | 65,536     | 65,534          |
| /20  | 255.255.240.0   | 12          | 4,096      | 4,094           |
| /21  | 255.255.248.0   | 11          | 2,048      | 2,046           |
| /23  | 255.255.254.0   | 9           | 512        | 510             |
| /24  | 255.255.255.0   | 8           | 256        | 254             |
| /25  | 255.255.255.128 | 7           | 128        | 126             |
| /26  | 255.255.255.192 | 6           | 64         | 62              |
| /27  | 255.255.255.224 | 5           | 32         | 30              |
| /28  | 255.255.255.240 | 4           | 16         | 14              |
| /29  | 255.255.255.248 | 3           | 8          | 6               |
| /30  | 255.255.255.252 | 2           | 4          | 2               |
| /32  | 255.255.255.255 | 0           | 1          | 1 (단일 호스트) |

### 서브넷 계산 예시

#### /20 네트워크 — 10.120.0.0/20

```
Network:   10.120.0.0
Broadcast: 10.120.15.255
Host range: 10.120.0.1 ~ 10.120.15.254
Total IPs: 4,096  |  Usable: 4,094

Bit structure:
Octet:  1         2         3         4
Value: 10  .    120  .     0   .     0
Bin: 00001010.01111000.0000|0000.00000000
                        ^^^^
                        /20 boundary (top 20 bits = network)

3rd octet range: 0 ~ 15 (lower 4 bits variable)
```

#### /24 네트워크 — 192.168.1.0/24

```
Network:    192.168.1.0
Broadcast:  192.168.1.255
Host range: 192.168.1.1 ~ 192.168.1.254
Usable:     254
```

#### /30 네트워크 — Point-to-Point

```
Network:    10.0.0.0
Broadcast:  10.0.0.3
Host range: 10.0.0.1 ~ 10.0.0.2
Usable:     2
Usage:      router-to-router link
```

## 5. CIDR 표기법

### 개념

```
Format: IP/prefix-length
Example: 192.168.1.0/24

/8  → 16,777,216 IPs
/16 →     65,536 IPs
/24 →        256 IPs
/28 →         16 IPs
/30 →          4 IPs
/32 →          1 IP  (single host)
```

### 서브넷 분할 예시

```
Original: 192.168.1.0/24  (254 hosts)

Split into 4 subnets (/26):
├── 192.168.1.0/26   (192.168.1.1   ~ 192.168.1.62)
├── 192.168.1.64/26  (192.168.1.65  ~ 192.168.1.126)
├── 192.168.1.128/26 (192.168.1.129 ~ 192.168.1.190)
└── 192.168.1.192/26 (192.168.1.193 ~ 192.168.1.254)

Each subnet: 62 usable hosts
```

### 서브넷 분할 공식

```
필요한 서브넷 수 → 추가 비트 수 (log2)

2개  → 1비트  (/24 → /25)
4개  → 2비트  (/24 → /26)
8개  → 3비트  (/24 → /27)
16개 → 4비트  (/24 → /28)
```

## 6. AWS VPC 서브넷

### AWS 예약 IP 규칙

AWS VPC의 모든 서브넷은 **5개 IP를 자동 예약**한다.

```
Example: 10.0.1.0/24

┌─────────────┬──────────────────────┐
│ IP          │ 용도                 │
├─────────────┼──────────────────────┤
│ 10.0.1.0    │ Network address      │
│ 10.0.1.1    │ VPC router           │
│ 10.0.1.2    │ DNS server           │
│ 10.0.1.3    │ Reserved (future)    │
│ 10.0.1.255  │ Broadcast            │
└─────────────┴──────────────────────┘

Usable: 256 - 5 = 251
```

### CIDR별 실사용 가능 IP

| CIDR | 전체 IP | AWS 예약 | 실사용 가능 | 비고             |
|------|---------|----------|-------------|------------------|
| /18  | 16,384  | 5        | 16,379      | ✅ 대규모 VPC    |
| /20  | 4,096   | 5        | 4,091       | ✅ 중규모 VPC    |
| /24  | 256     | 5        | 251         | ✅ 일반적        |
| /28  | 16      | 5        | 11          | ✅ AWS 권장 최소 |
| /29  | 8       | 5        | 3           | ⚠️ 매우 제한적    |
| /30  | 4       | 5        | 0           | ❌ 사용 불가     |

### /30 사용 불가 이유

```
10.0.0.0/30 — Total 4 IPs

┌────────────┬──────────────────────────┐
│ IP         │ AWS reserved             │
├────────────┼──────────────────────────┤
│ 10.0.0.0   │ Network address          │
│ 10.0.0.1   │ VPC router               │
│ 10.0.0.2   │ DNS server               │
│ 10.0.0.3   │ Reserved + Broadcast     │
└────────────┴──────────────────────────┘

All 4 IPs reserved → no hosts assignable
```

⚠️ AWS VPC 서브넷 최소 권장 크기: **/28** (실사용 11개)

### 10.120.0.0/20 서브넷 분할 예시

```
VPC CIDR: 10.120.0.0/20
Range:    10.120.0.0 ~ 10.120.15.255
Total:    4,096 IPs  |  Usable: 4,091

Subnet split example (/24):
├── 10.120.0.0/24   Public  - AZ-a
├── 10.120.1.0/24   Public  - AZ-b
├── 10.120.2.0/24   Public  - AZ-c
├── 10.120.3.0/24   Private - AZ-a
├── 10.120.4.0/24   Private - AZ-b
└── 10.120.5.0/24   Private - AZ-c
```

## 7. 실전 예시

### IP 주소 판별

```
IP: 172.16.5.100

1. Class:   1st octet 172 (128~191) → Class B
2. Type:    172.16.0.0 ~ 172.31.255.255 → Private IP
3. Mask:    Class B default → 255.255.0.0 (/16)
4. Network: 172.16.0.0
5. Host:    5.100
```

### 보안 그룹 CIDR 확인

```
Allow CIDR: 203.0.112.0/23
Range:      203.0.112.0 ~ 203.0.113.255

203.0.113.55  → Allow  (in range)
123.111.112.1 → Deny   (out of range)
```

## 8. 계산 도구

### Linux 명령어

```bash
# ipcalc
ipcalc 192.168.1.100/24

# ip 명령어로 인터페이스 확인
ip addr show
```

### Python 스크립트

```python
import ipaddress

network = ipaddress.ip_network('192.168.1.0/24')
print(network.network_address)   # 192.168.1.0
print(network.broadcast_address) # 192.168.1.255
print(network.netmask)           # 255.255.255.0
print(network.num_addresses - 2) # 254

ip = ipaddress.ip_address('192.168.1.100')
print(ip in network)  # True
```

### 온라인 도구

| 도구              | URL                                |
|-------------------|------------------------------------|
| Subnet Calculator | https://www.subnet-calculator.com/ |
| CIDR.xyz          | https://cidr.xyz/                  |

### 핵심 공식

```
호스트 수       = 2^(호스트 비트) - 2
서브넷 분할     = 기존 프리픽스 + log2(필요한 서브넷 수)
AWS 실사용 IP   = 전체 IP - 5
```
