## IP 주소 기본 구조

### IPv4 주소 구성

```
192.168.1.100
├─ 옥텟1: 192
├─ 옥텟2: 168
├─ 옥텟3: 1
└─ 옥텟4: 100

각 옥텟: 0 ~ 255 (8비트 = 1바이트)
전체: 32비트 (4바이트)
```

### 네트워크 영역 vs 호스트 영역

```
IP 주소 = 네트워크 영역 + 호스트 영역

192.168.1.100/24
├─ 네트워크 영역: 192.168.1 (네트워크 식별)
└─ 호스트 영역: 100 (호스트 식별)
```

**역할:**
- **네트워크 영역**: 어느 네트워크에 속하는지 식별
- **호스트 영역**: 네트워크 내에서 특정 장치 식별

**용어 어원:**
- **네트워크 (Network)**: "net(그물)" + "work(작업)" → 그물망처럼 연결된 장치들
- **호스트 (Host)**: "host(주인)" → 네트워크에 연결된 개별 장치 (1960년대 메인프레임 컴퓨터를 "host computer"라 부른 것에서 유래)

**비유:**
```
192.168.1.100/24
├─ 네트워크 (192.168.1): 서울시 강남구 (동네)
└─ 호스트 (100): 123번지 (집 주소)

라우터는 "강남구"만 보고 방향 결정
강남구 내부에서 "123번지" 찾기
```

## 클래스별 IP 주소

### Class A (대규모 네트워크)

```
범위: 0.0.0.0 ~ 127.255.255.255
첫 비트: 0xxxxxxx

기본 서브넷 마스크: 255.0.0.0 (/8)

구조:
N.H.H.H
└─ N: 네트워크 (8비트)
   H: 호스트 (24비트)

예시: 10.0.0.0/8
├─ 네트워크 수: 126개 (2^7 - 2, 0과 127 예약)
├─ 호스트 수: 16,777,214개 (2^24 - 2)
└─ 용도: 초대규모 조직
```

**특징:**
- 네트워크 수: 적음 (126개, 0과 127 예약)
- 호스트 수: 매우 많음 (1,600만 개)
- 사용 예: 대기업, ISP

**사설 IP 대역:**
- `10.0.0.0 ~ 10.255.255.255` (10.0.0.0/8)

### Class B (중규모 네트워크)

```
범위: 128.0.0.0 ~ 191.255.255.255
첫 비트: 10xxxxxx

기본 서브넷 마스크: 255.255.0.0 (/16)

구조:
N.N.H.H
└─ N: 네트워크 (16비트)
   H: 호스트 (16비트)

예시: 172.16.0.0/16
├─ 네트워크 수: 16,384개 (2^14)
├─ 호스트 수: 65,534개 (2^16 - 2)
└─ 용도: 중규모 조직
```

**특징:**
- 네트워크 수: 중간 (16,384개)
- 호스트 수: 많음 (65,534개)
- 사용 예: 중견기업, 대학

**사설 IP 대역:**
- `172.16.0.0 ~ 172.31.255.255` (172.16.0.0/12)

### Class C (소규모 네트워크)

```
범위: 192.0.0.0 ~ 223.255.255.255
첫 비트: 110xxxxx

기본 서브넷 마스크: 255.255.255.0 (/24)

구조:
N.N.N.H
└─ N: 네트워크 (24비트)
   H: 호스트 (8비트)

예시: 192.168.1.0/24
├─ 네트워크 수: 2,097,152개 (2^21)
├─ 호스트 수: 254개 (2^8 - 2)
└─ 용도: 소규모 조직
```

**특징:**
- 네트워크 수: 많음 (200만 개)
- 호스트 수: 적음 (254개)
- 사용 예: 소기업, 가정

**사설 IP 대역:**
- `192.168.0.0 ~ 192.168.255.255` (192.168.0.0/16)

### Class D (멀티캐스트)

```
범위: 224.0.0.0 ~ 239.255.255.255
첫 비트: 1110xxxx

용도: 멀티캐스트 그룹 주소
예시: 224.0.0.1 (모든 호스트)
```

### Class E (예약)

```
범위: 240.0.0.0 ~ 255.255.255.255
첫 비트: 1111xxxx

용도: 실험 및 연구용 (일반 사용 불가)
```

### 클래스 비교표

| 클래스 | 범위 | 기본 마스크 | 네트워크 비트 | 호스트 비트 | 네트워크 수 | 호스트 수 | 용도 |
|--------|------|------------|--------------|------------|------------|----------|------|
| **A** | 0.0.0.0 ~ 127.255.255.255 | 255.0.0.0 (/8) | 8 | 24 | 126 | 16,777,214 | 대규모 |
| **B** | 128.0.0.0 ~ 191.255.255.255 | 255.255.0.0 (/16) | 16 | 16 | 16,384 | 65,534 | 중규모 |
| **C** | 192.0.0.0 ~ 223.255.255.255 | 255.255.255.0 (/24) | 24 | 8 | 2,097,152 | 254 | 소규모 |
| **D** | 224.0.0.0 ~ 239.255.255.255 | - | - | - | - | - | 멀티캐스트 |
| **E** | 240.0.0.0 ~ 255.255.255.255 | - | - | - | - | - | 예약 |

## 서브넷 마스크

```
서브넷 마스크: 네트워크 영역과 호스트 영역을 구분

        1 (네트워크)              0 (호스트)
```

## 특수 IP 주소

### 네트워크 주소

```
정의: 호스트 영역이 모두 0인 주소
용도: 네트워크 자체를 식별

예시:
192.168.1.0/24 → 네트워크 주소
10.0.0.0/16 → 네트워크 주소

특징: 호스트에 할당 불가
```

### 브로드캐스트 주소

```
정의: 호스트 영역이 모두 1인 주소
용도: 네트워크 내 모든 호스트에게 전송

예시:
192.168.1.255/24 → 브로드캐스트
10.0.255.255/16 → 브로드캐스트

특징: 호스트에 할당 불가
```

### 루프백 주소

```
범위: 127.0.0.0 ~ 127.255.255.255
대표: 127.0.0.1 (localhost)

용도: 자기 자신을 가리킴
예시: ping 127.0.0.1 → 자신에게 핑
```

### 사설 IP (Private IP)

```
Class A: 10.0.0.0 ~ 10.255.255.255 (10.0.0.0/8)
Class B: 172.16.0.0 ~ 172.31.255.255 (172.16.0.0/12)
Class C: 192.168.0.0 ~ 192.168.255.255 (192.168.0.0/16)

특징:
- 인터넷에서 라우팅 불가
- NAT 필요
- 무료 사용 가능
```

### 공인 IP (Public IP)

```
정의: 인터넷에서 라우팅 가능한 IP
특징:
- 전 세계에서 유일
- ISP에서 할당
- 비용 발생

예시: 198.51.100.10 (AWS EC2)
```

### APIPA (자동 사설 IP)

```
범위: 169.254.0.0 ~ 169.254.255.255 (169.254.0.0/16)

용도: DHCP 실패 시 자동 할당
예시: 169.254.1.1

의미: 네트워크 설정 문제
```

### 링크 로컬 주소

```
범위: 169.254.0.0/16 (IPv4)
     fe80::/10 (IPv6)

용도: 로컬 네트워크 통신
특징: 라우터 통과 불가
```

## CIDR 표기법

### CIDR (Classless Inter-Domain Routing)

```
형식: IP주소/프리픽스길이
예시: 192.168.1.0/24

장점:
- 클래스 제약 없음
- 유연한 서브넷 분할
- IP 주소 효율적 사용
```

### CIDR 블록 예시

```
/8  → 16,777,216개 IP (Class A 1개)
/16 → 65,536개 IP (Class B 1개)
/24 → 256개 IP (Class C 1개)
/25 → 128개 IP (Class C 절반)
/26 → 64개 IP (Class C 1/4)
/27 → 32개 IP
/28 → 16개 IP
/29 → 8개 IP
/30 → 4개 IP (P2P 연결)
/32 → 1개 IP (단일 호스트)
```

### 서브넷 분할 예시

```
원본: 192.168.1.0/24 (254개 호스트)

4개로 분할 (/26):
├─ 192.168.1.0/26   (192.168.1.1 ~ 62)
├─ 192.168.1.64/26  (192.168.1.65 ~ 126)
├─ 192.168.1.128/26 (192.168.1.129 ~ 190)
└─ 192.168.1.192/26 (192.168.1.193 ~ 254)

각 서브넷: 62개 호스트
```

## 실전 예시

### AWS VPC 설계

```
VPC CIDR: 10.0.0.0/16 (65,536개 IP)

서브넷 분할:
├─ Public Subnet A:  10.0.1.0/24 (254개)
├─ Public Subnet B:  10.0.2.0/24 (254개)
├─ Public Subnet C:  10.0.3.0/24 (254개)
├─ Private Subnet A: 10.0.10.0/24 (254개)
├─ Private Subnet B: 10.0.20.0/24 (254개)
└─ Private Subnet C: 10.0.30.0/24 (254개)

예약 IP (각 서브넷):
├─ 10.0.1.0: 네트워크 주소
├─ 10.0.1.1: VPC 라우터
├─ 10.0.1.2: DNS 서버
├─ 10.0.1.3: AWS 예약
└─ 10.0.1.255: 브로드캐스트

실제 사용 가능: 251개 (254 - 3)
```

### 보안 그룹 설정

```
허용 CIDR: 203.0.112.0/23

포함 범위:
├─ 203.0.112.0 ~ 203.0.112.255
└─ 203.0.113.0 ~ 203.0.113.255

테스트:
203.0.113.55 → 허용 (범위 내)
123.111.112.1  → 차단 (범위 외)
```

### 서브넷 계산 실습

```
문제: 192.168.10.0/24를 4개 서브넷으로 분할

해결:
1. 필요한 서브넷: 4개 → 2^2 = 4
2. 추가 비트: 2비트
3. 새 프리픽스: /24 + 2 = /26
4. 각 서브넷: 2^6 = 64개 IP (62개 사용 가능)

결과:
├─ 192.168.10.0/26   (0 ~ 63)
├─ 192.168.10.64/26  (64 ~ 127)
├─ 192.168.10.128/26 (128 ~ 191)
└─ 192.168.10.192/26 (192 ~ 255)
```

### IP 주소 판별

```
IP: 172.16.5.100

1. 클래스 판별:
   첫 옥텟: 172 (128~191)
   → Class B

2. 사설/공인 판별:
   172.16.0.0 ~ 172.31.255.255
   → 사설 IP

3. 기본 서브넷:
   Class B → 255.255.0.0 (/16)

4. 네트워크/호스트:
   네트워크: 172.16
   호스트: 5.100
```

## 계산 도구

### Linux 명령어

```bash
# ipcalc 사용
ipcalc 192.168.1.100/24

# 출력:
# Address:   192.168.1.100
# Netmask:   255.255.255.0 = 24
# Network:   192.168.1.0/24
# HostMin:   192.168.1.1
# HostMax:   192.168.1.254
# Broadcast: 192.168.1.255
# Hosts/Net: 254
```

### Python 스크립트

```python
import ipaddress

# 네트워크 정보
network = ipaddress.ip_network('192.168.1.0/24')

print(f"네트워크: {network.network_address}")
print(f"브로드캐스트: {network.broadcast_address}")
print(f"넷마스크: {network.netmask}")
print(f"호스트 수: {network.num_addresses - 2}")
print(f"첫 호스트: {list(network.hosts())[0]}")
print(f"마지막 호스트: {list(network.hosts())[-1]}")

# IP 포함 여부 확인
ip = ipaddress.ip_address('192.168.1.100')
print(f"{ip} in network: {ip in network}")
```

## 요약 체크리스트

### IP 주소 분석 순서

```
1. 클래스 확인
   └─ 첫 옥텟으로 판별 (A: 0~127, B: 128~191, C: 192~223)

2. 사설/공인 확인
   └─ 10.x.x.x, 172.16~31.x.x, 192.168.x.x → 사설

3. 서브넷 마스크 확인
   └─ CIDR 표기 또는 10진수 표기

4. 네트워크 주소 계산
   └─ IP AND 서브넷 마스크

5. 브로드캐스트 주소 계산
   └─ 네트워크 주소 + (2^호스트비트 - 1)

6. 사용 가능 IP 범위
   └─ 네트워크주소+1 ~ 브로드캐스트-1
```

### 핵심 공식

```
호스트 수 = 2^호스트비트 - 2
네트워크 수 = 2^네트워크비트

사용 가능 IP = 전체 IP - 2 (네트워크 + 브로드캐스트)

서브넷 분할:
새 프리픽스 = 기존 프리픽스 + log2(필요한 서브넷 수)
```

### log2 이해하기

**log2(x) = "2를 몇 번 곱해야 x가 되는가?"**

```
log2(2) = 1    → 2^1 = 2
log2(4) = 2    → 2^2 = 4
log2(8) = 3    → 2^3 = 8
log2(16) = 4   → 2^4 = 16
```

**서브넷 분할 적용:**

```
필요한 서브넷 → 필요한 비트

2개 필요  → log2(2) = 1비트
4개 필요  → log2(4) = 2비트
8개 필요  → log2(8) = 3비트
16개 필요 → log2(16) = 4비트
```

**예시: 192.168.1.0/24를 4개로 분할**

```
1. 필요한 서브넷: 4개
2. 필요한 비트: log2(4) = 2비트
3. 새 프리픽스: /24 + 2 = /26

결과: 4개의 /26 서브넷 (각 64개 IP)
```

**빠른 참조:**

| 서브넷 수 | log2 | 비트 | 계산 |
|----------|------|------|------|
| 2 | 1 | 1비트 | 2^1 = 2 |
| 4 | 2 | 2비트 | 2^2 = 4 |
| 8 | 3 | 3비트 | 2^3 = 8 |
| 16 | 4 | 4비트 | 2^4 = 16 |
