## 1. 개요

경계값 분석(BVA)은 입력 범위의 경계에서 결함이 발생할 확률이 높다는 경험적 사실에 기반한 테스트 기법입니다.

| 항목          | 내용                                         |
|---------------|----------------------------------------------|
| 분류          | 블랙박스 테스트 기법                         |
| 기반          | 동등 분할(Equivalence Partitioning)의 보완   |
| 핵심 아이디어 | 경계 ±1 지점에서 버그가 집중 발생            |
| 적용 대상     | 숫자 범위, 배열 인덱스, 문자열 길이, 날짜 등 |

[⬆ 목차로 돌아가기](#목차)

## 2. 원리

입력 범위 `[min, max]`에 대해 경계와 그 인접 값을 테스트합니다.

```
        Invalid    │      Valid         │    Invalid
  ─────────────────┼────────────────────┼─────────────────
                min-1  min  min+1 ... max-1  max  max+1
                  ↑     ↑    ↑          ↑     ↑    ↑
            Test target values
```

### 왜 경계에서 버그가 발생하는가

- `<` vs `<=` 혼동 (off-by-one error)
- 배열 인덱스 0 시작 vs 1 시작
- 반복문 종료 조건 오류
- 타입 오버플로우 (MAX_INT + 1)

## 3. 테스트 값 선정

### 2-value BVA (기본)

경계값과 바로 바깥 값만 테스트합니다.

| 위치      | 테스트 값 | 기대 결과     |
|-----------|-----------|---------------|
| 하한 경계 | `min`     | 유효          |
| 하한 바깥 | `min - 1` | 유효하지 않음 |
| 상한 경계 | `max`     | 유효          |
| 상한 바깥 | `max + 1` | 유효하지 않음 |

### 3-value BVA (강화)

경계값, 바로 안쪽, 바로 바깥 값을 테스트합니다.

| 위치      | 테스트 값 | 기대 결과     |
|-----------|-----------|---------------|
| 하한 바깥 | `min - 1` | 유효하지 않음 |
| 하한 경계 | `min`     | 유효          |
| 하한 안쪽 | `min + 1` | 유효          |
| 상한 안쪽 | `max - 1` | 유효          |
| 상한 경계 | `max`     | 유효          |
| 상한 바깥 | `max + 1` | 유효하지 않음 |

## 4. 예시

### 예시 1: 나이 입력 (0~150)

```
유효 범위: 0 ≤ age ≤ 150

테스트 값: -1, 0, 1, 149, 150, 151
```

| 입력 | 기대 결과 | 검증 포인트 |
|------|-----------|-------------|
| -1   | 에러      | 하한 바깥   |
| 0    | 성공      | 하한 경계   |
| 1    | 성공      | 하한 안쪽   |
| 149  | 성공      | 상한 안쪽   |
| 150  | 성공      | 상한 경계   |
| 151  | 에러      | 상한 바깥   |

### 예시 2: 비밀번호 길이 (8~20자)

| 입력 길이 | 기대 결과 | 검증 포인트 |
|-----------|-----------|-------------|
| 7         | 에러      | 하한 바깥   |
| 8         | 성공      | 하한 경계   |
| 9         | 성공      | 하한 안쪽   |
| 19        | 성공      | 상한 안쪽   |
| 20        | 성공      | 상한 경계   |
| 21        | 에러      | 상한 바깥   |

### 예시 3: 배열 인덱스 (0~N-1)

배열 크기 N=10인 경우:

| 인덱스 | 기대 결과  | 검증 포인트 |
|--------|------------|-------------|
| -1     | IndexError | 하한 바깥   |
| 0      | 정상 접근  | 하한 경계   |
| 1      | 정상 접근  | 하한 안쪽   |
| 8      | 정상 접근  | 상한 안쪽   |
| 9      | 정상 접근  | 상한 경계   |
| 10     | IndexError | 상한 바깥   |

### 예시 4: 날짜 (월별 일수)

2월 (평년):

| 입력 | 기대 결과   | 검증 포인트      |
|------|-------------|------------------|
| 0    | 에러        | 하한 바깥        |
| 1    | 성공        | 하한 경계        |
| 28   | 성공        | 상한 경계 (평년) |
| 29   | 에러 (평년) | 상한 바깥 (평년) |
| 29   | 성공 (윤년) | 상한 경계 (윤년) |
| 30   | 에러 (윤년) | 상한 바깥 (윤년) |

## 5. 확장 기법

### 다중 변수 경계값

2개 이상 입력이 있을 때, 한 변수는 경계값으로 고정하고 나머지는 정상값으로 설정합니다.

```
입력: x (1~100), y (1~50)

테스트 케이스:
  (1, 25)    ← x 하한, y 정상
  (100, 25)  ← x 상한, y 정상
  (50, 1)    ← x 정상, y 하한
  (50, 50)   ← x 정상, y 상한
```

### 특수 경계값

| 타입       | 경계값                                   |
|------------|------------------------------------------|
| 정수       | 0, -1, 1, MAX_INT, MIN_INT               |
| 부동소수점 | 0.0, -0.0, NaN, Infinity, 매우 작은 양수 |
| 문자열     | 빈 문자열, 1자, 최대 길이, NULL          |
| 컬렉션     | 빈 배열, 1개, 최대 크기                  |
| 날짜       | 윤년 2/29, 월말, 연말, epoch             |

### 동등 분할과의 관계

```
동등 분할: 범위를 그룹으로 나눔 → 각 그룹에서 대표값 1개 선택
경계값 분석: 그룹 경계에서 값 선택 → 동등 분할을 보완

함께 사용하면 테스트 커버리지가 높아집니다.
```
