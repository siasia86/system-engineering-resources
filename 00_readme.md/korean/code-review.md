전체 코드 리뷰 해줘.

## 필수 점검 항목

### 1. 잠재적 버그
- 런타임 에러 (None 참조, 인덱스 초과, 타입 불일치)
- 엣지 케이스 (빈 값, 0, 빈 리스트, 경계값)
- 경쟁 조건 (파일 락, 동시 실행)

### 2. 잘못된 설계
- 책임 분리 (하나의 함수가 너무 많은 일)
- 의존성 (전역 변수 남용, 순환 참조)
- 확장성 (하드코딩, 매직 넘버)

### 3. 의도치 않은 동작
- 부작용 (예상 외 파일 삭제/수정)
- 순서 의존 (실행 순서에 따라 결과 달라짐)
- 암묵적 상태 변경 (환경 변수, 전역 상태)

### 4. 에러 처리
- 예외 처리 누락 (try/except 범위, bare except)
- 리턴값 미검증 (subprocess, API 호출)
- 리소스 해제 (파일 핸들, 락, 소켓)

### 5. 보안
- 입력 검증 부재 (경로 조작, 인젝션)
- 권한 문제 (파일 퍼미션, 실행 권한)
- 민감 정보 노출 (로그, 에러 메시지)

### 6. 성능
- 불필요한 I/O (반복 파일 읽기, 중복 API 호출)
- 메모리 (대용량 파일 전체 로드, 리스트 누적)
- 비효율 루프 (중첩 루프, 불필요한 정렬)

### 7. 주석/문서 불일치
- docstring과 실제 동작 불일치
- 설정 키 누락 (config 문서)
- 에러 코드/상수 불일치

결과를 항목별 ✅ ❌ ⚠️ 로 표시하고, 수정 제안은 코드 블록으로 보여줘.
